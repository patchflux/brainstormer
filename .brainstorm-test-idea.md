# Test Idea for Brainstorm Validation

## The Idea

A CLI tool that helps developers write better commit messages by analyzing their staged changes and suggesting conventional commit format messages. It would:

- Read git diff of staged files
- Understand the type of change (feat, fix, refactor, docs, etc.)
- Generate a concise commit message
- Support customization via config file

## Why This Idea

- Common pain point (developers write bad commit messages)
- Clear scope (small CLI tool)
- Technical feasibility obvious
- No external API dependencies needed for MVP
